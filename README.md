# BattleFPS

Apexライクなブラウザ対戦FPS。ルームを作成して友達と複数人でバトルロイヤルが遊べます。
HTML + Three.js（クライアント）と Node.js + WebSocket（サーバー）だけで動作します。

## 特徴

- 🎮 **ブラウザだけで動く3D FPS** — インストール不要。Three.js でレンダリング
- 🚪 **ルーム制マルチプレイ** — 4桁コードでルームを作成／参加
- 🔫 **複数の武器** — ピストル / アサルトライフル / ショットガン / スナイパー（弾種・連射・リロード・射程が異なる）
- 🎒 **インベントリ & ルート** — マップに落ちた武器・弾薬・回復・シールドを `E` で拾う。撃破するとアイテムをドロップ
- 🛡️ **シールド（アーマー）** — ダメージはシールドが先に吸収。セル / バッテリーでチャージ
- 💉 **回復アイテム** — 注射器・メドキット（HP）／シールドセル・バッテリー（シールド）
- 🟦 **縮小する安全地帯（リング）** — Apexのように時間経過で縮小、外にいるとダメージ
- 🏆 **バトルロイヤル** — 最後の1人になったら `#1 VICTORY ROYALE`
- 🗺️ ミニマップ / キルフィード / HP・シールド・武器・残弾HUD / テキストチャット

## 動かし方

```bash
npm install      # ws をインストール
npm start        # サーバー起動（デフォルト http://localhost:3000）
```

ブラウザで `http://localhost:3000` を開きます。
複数人で遊ぶには、同じネットワーク／公開URL上のサーバーに各自アクセスしてください。

ポートを変える場合: `PORT=8080 npm start`

### 一人でテストする

1. ブラウザでサーバーを開き、名前を入れて「ルームを作成」
2. 表示された4桁コードを別タブ／別ブラウザ／別端末で「参加」に入力
3. ホスト（作成者）が「ゲーム開始」を押す（1人でも開始可能）

## 操作方法

| 操作 | キー |
|------|------|
| 移動 | `W` `A` `S` `D` |
| スプリント | `Shift` |
| ジャンプ | `Space` |
| 視点 | マウス（画面クリックでマウスロック） |
| 射撃 | 左クリック（押しっぱなしで連射） |
| 武器切替 | `1` `2` `3` / マウスホイール |
| リロード | `R` |
| アイテムを拾う | `E` |
| 回復（HP） | `Z` 注射器 / `C` メドキット |
| シールド回復 | `X` セル / `V` バッテリー |
| チャット | `Enter` |

## 武器

| 武器 | 弾種 | ダメージ | マガジン | 特徴 |
|------|------|----------|----------|------|
| ピストル | ライト | 20 | 12 | 初期装備・万能 |
| アサルトライフル | ライト | 15 | 30 | 高い連射・中射程 |
| ショットガン | シェル | 11×9 | 6 | 近距離で高火力 |
| スナイパー | ヘビー | 65 | 4 | 長射程・高威力 |

ヘッドショットは武器ごとの倍率でダメージが増加します。

## アイテム画像（ChatGPT Images 2.0）

アイテムのアイコン画像は **ChatGPT の Images 2.0（`gpt-image-1`）** で生成して差し込めます。

1. [`ITEM_IMAGE_PROMPTS.md`](./ITEM_IMAGE_PROMPTS.md) をChatGPTに貼り付けて画像を生成
2. 出力された透過PNGを `public/assets/items/` に指定ファイル名で保存
3. ゲームが自動で読み込み、地面のアイテムをそのアイコンで表示します
   （画像が無いキーはプロシージャルな3D形状にフォールバックするので、未配置でも動作します）

## 構成

```
server/server.js      WebSocketサーバー + 静的配信 + ルーム/ゲーム状態（武器/シールド/ルート/回復）
public/index.html     メニュー・ロビー・HUD
public/css/style.css  UIスタイル
public/js/shared.js   武器/回復/ルートの共通定義（サーバー・クライアント共用）
public/js/net.js      WebSocketクライアントラッパー
public/js/game.js     Three.js FPSエンジン（マップ/移動/射撃/武器/ルート/リング/ミニマップ）
public/js/main.js     UI と ネットワークとゲームの接続
public/assets/items/  アイテム画像（ChatGPTで生成して配置）
ITEM_IMAGE_PROMPTS.md ChatGPT Images 2.0 用の画像生成指示書
```

サーバーは20Hzで全プレイヤーの状態をブロードキャストします。命中判定はクライアントが
報告し、サーバーが武器射程で簡易検証してダメージ（シールド→HPの順）を適用する方式です。
